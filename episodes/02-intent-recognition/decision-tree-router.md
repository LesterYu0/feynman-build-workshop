# 资产 #3：路由决策树

> 每层选什么技术、不选什么、为什么——做技术决策时逐项对照。

---

## 根节点：你的路由目标是什么？

```
┌─────────────────────────────────────┐
│  目标 ≠ 最高准确率                     │
│  目标 = min(延迟 × 成本 × 错误率)       │
└──────────────┬──────────────────────┘
               │
               ▼
```

## L0：关键词 / 正则 / Trie

| 选什么 | 不选什么 | 为什么 |
|------|------|------|
| `re.compile` 模块级预编译 | `re.match` 函数内每次编译 | 1000QPS 时 CPU 差 200 倍 |
| Trie 前缀树加速命令匹配 | 暴力遍历所有正则 | >50 条规则时 Trie 明显优势 |
| 简单 `r'^/command'` 模式 | 复杂正则回溯 | 覆盖 60%+ 流量即可，不追求 100% |

### L0 决策检查

- [ ] 所有正则预编译在模块顶层
- [ ] 规则按频率排序（高频规则放前面）
- [ ] >50 条规则时加上 Trie 加速
- [ ] L0 命中后直接返回，不走后续层

---

## L1：向量语义路由

| 选什么 | 不选什么 | 为什么 |
|------|------|------|
| `semantic-router` (Python) | 手写向量检索 | 开箱即用，维护成本低 |
| `bge-small-zh` (中文) | `all-MiniLM-L6` (英文) | 中文场景准确率高 15%+ |
| 每意图 5-20 条例句 | 每意图 1-2 条 | 1 条覆盖的语义空间太小 |
| 余弦相似度 | 欧氏距离 | embedding 归一化后余弦更稳定 |

### L1 决策检查

- [ ] 每个意图有 5+ 例句，覆盖不同表达方式
- [ ] 例句包含真实用户说法（不是改写）
- [ ] `threshold` 按业务风险调整（退款=0.7，天气=0.5）
- [ ] 编码器只加载一次，不复用

## L2：大模型 FC + 结构化输出

| 选什么 | 不选什么 | 为什么 |
|------|------|------|
| `response_format: json_schema` + `strict: true` | 自由文本 + 正则后处理 | 数学保证不输出幻觉意图名 |
| `temperature = 0` | `temperature > 0` | 同样输入→同样输出，分类必须确定 |
| Enum 约束 | 字符串无约束 | 输出只能是预定义值 |
| 槽位填充 + 追问 | 缺失槽位就猜/忽略 | 猜错=执行失败，追问=零风险 |

### L2 决策检查

- [ ] temperature 永久设为 0
- [ ] 输出用 Enum 约束，不用 free-text
- [ ] 缺失槽位时反问，不猜
- [ ] active_slot 检测、取消检测、上下文继承全部实现

## Safe Net：安全网

| 选什么 | 不选什么 | 为什么 |
|------|------|------|
| 置信度分级 (≥0.8/0.6-0.8/<0.6) | 单一阈值 | 一刀切太低→误判多，一刀切太高→L2 调用太多 |
| OOD: 全意图 max < 0.4 | OOD: 硬分类到「其他」 | 其他类会吸收奇奇怪怪的输入 |
| Session 级熔断 (Hystrix 模式) | 全局熔断 | 全局熔断伤及无辜用户 |
| 连续 3 次低置信→关 L2 × 5 轮 | 永远开 L2 | 一个人讲笑话=其他人的 LLM 调用被浪费 |

### Safe Net 决策检查

- [ ] OOD 检测阈值 < 0.4
- [ ] 熔断粒度=session 级，非全局
- [ ] 熔断后 5 轮自动恢复（half-open 试探）
- [ ] 可观测：每次 L2 调用都记录意图/置信/latency

---

## 可观测性

| 指标 | 为什么不是普通健康检查 | 告警阈值 |
|------|------|:--:|
| 错误路由率 | 路由失败=200 但意图错 | > 5%/天 |
| P95 延迟 | 太多请求穿透到 L2 | > 300ms |
| 意图分布 KL 散度 | 业务变化但路由没跟 | > 20% 日环比 |
