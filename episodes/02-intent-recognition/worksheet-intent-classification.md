# 资产 #1：意图分类工作表

## 使用说明

在给你的 Agent 设计意图路由之前，先回答以下 5 个问题。这份工作表帮你避免「识别了 100 种意图，但 80% 的请求不在里面」的经典错误。

---

## 第 1 步：列出你的 Agent 要处理的 TOP 20 任务

> 不是「可能」要处理的——是最近一周日志里真实出现频率 TOP 20。

| 排名 | 用户说什么（原文/典型说法） | 意图 ID | 频率 |
|:----:|------|------|:----:|
| 1 | | | % |

| ... | | | |
| 20 | | | % |

✅ 检查：TOP 20 覆盖了总请求的多少 %？目标 > 80%。

## 第 3 步：为 L1 准备例句（每意图 5-20 条）

> 例句必须覆盖同意图的不同表达方式——不是改写，是真实用户会说的人话。

示例：

## 交付检查

- [ ] TOP 20 意图覆盖 > 80% 真实流量
- [ ] 每个 L1 意图都有 5+ 条例句，覆盖不同表达
