# 贡献指南

感谢你考虑为造物车间贡献改进！

## 如何贡献

### 🐛 报告 Bug 或问题

如果你在使用某期资产时遇到问题（代码跑不通、测试用例不合理、决策树有遗漏），直接在对应视频评论区反馈，或者在 GitHub Issues 里描述：

- 哪一期、哪个文件
- 你做了什么操作
- 预期的结果 vs 实际的结果

### ✨ 改进现有资产

最欢迎的贡献类型：

1. **代码补丁**：修复 bug、添加边界条件、改进性能
2. **新增测试用例**：为你踩过的坑补充验证点
3. **决策树更新**：补充新的技术选型维度或修正过时结论
4. **文档改进**：补充使用说明、踩坑记录

### 🆕 贡献新资产

如果你想基于某期视频补充一份新资产（例如第 01 期加一份 Docker 部署模板），开 Issue 先聊一下思路，确认后发 PR。

## PR 流程

1. Fork 本仓库
2. 创建你的 feature 分支 (`git checkout -b feat/your-improvement`)
3. 提交改动 (`git commit -m 'feat: add xxx'`)
4. 推送到你的 fork (`git push origin feat/your-improvement`)
5. 开 Pull Request，描述你做了什么、为什么这样做

## 代码风格

- Python：遵循 PEP 8，类型标注
- Markdown：中文和英文之间加空格，代码块标注语言
- 文件命名：`类型-描述.ext` 格式（如 `code-agent-memory-core.py`）

## 行为准则

保持专业、友善。我们是一个帮助技术人把 AI 学明白的社区。
